Input and Output in Angular

Input:

Used to pass data from a parent component to a child component. The child component receives data from the parent via Input.

Output:

Used to emit events from a child component to a parent component. The child component uses Output with EventEmitter to notify the parent about changes or actions.

Why This Design?

Unidirectional Data Flow:

  • Angular enforces a unidirectional data flow:
    • Parent to Child via Input.
    • Child to Parent via Output.
  • This makes the data flow predictable and easier to debug.

Separation of Concerns:

  • The parent component manages the state and passes data to the child.
  • The child component notifies the parent about user interactions or changes.
